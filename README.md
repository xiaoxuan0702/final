## 一、前言（Introduction）

隨著大專院校學生社團的多元發展，社團數量與活動規模逐年增加，傳統以紙本、通訊軟體或零散工具進行管理的方式，已逐漸無法滿足現代社團在成員管理、活動規劃與資訊整合上的需求。社團幹部在管理過程中，常面臨資料分散、資訊不透明以及行政效率低落等問題。

因此，本專題以「學生社團管理系統」為主題，規劃並設計一套以 Web 為基礎的管理平台，提供社團成員、幹部與管理者一個集中且系統化的管理工具。透過本系統，使用者可以更方便地瀏覽社團資訊、參與活動，而社團幹部也能有效進行成員與活動管理。

本系統的目標在於提升社團行政效率、降低管理負擔，並促進校園社團運作的數位化與現代化。


## 二、系統概述（System Overview）

本系統為一套以 Web 為基礎的學生社團管理平台，目的在於整合校園內各類社團的管理需求，提供一個集中、便利且易於操作的數位化系統。透過瀏覽器即可使用，無需額外安裝應用程式，讓學生與社團幹部能隨時隨地進行社團相關操作。

系統主要使用對象包含以下三類角色：

- **一般學生（社團成員）**  
  可透過系統瀏覽社團列表、查看社團介紹與活動資訊，並進行社團加入申請與活動報名，清楚掌握自身的社團參與狀況。

- **社團幹部**  
  負責社團日常管理工作，包括成員審核、公告發布、活動建立與管理，以及活動報名名單的查看，藉此降低行政負擔並提升管理效率。

- **系統管理員**  
  擁有最高管理權限，負責維護系統運作、管理使用者帳號與社團資料，確保系統資料的正確性與安全性。

整體系統以「角色權限區分」為核心設計概念，不同身分的使用者僅能操作對應權限的功能，以確保資料安全與管理流程的清楚分工。透過本系統，校園社團的資訊管理將更加有系統性，並促進社團運作的效率與透明度。

## ⚙️ 功能需求分析（Functional Requirements）

### 使用者功能
- 使用者註冊與登入
- 瀏覽社團列表與社團詳細資訊
- 申請加入社團
- 報名社團活動
- 查詢個人參與紀錄

### 社團幹部功能
- 管理社團成員（審核加入、移除成員）
- 發布社團公告
- 建立與管理社團活動
- 查看活動報名名單

### 系統管理員功能
- 管理使用者帳號
- 管理社團資料
- 系統權限控管與維護

## 🧩 系統設計（System Design）
本系統採用前後端分離架構：
- **前端**：HTML、CSS、JavaScript
- **後端**：Node.js / PHP / Python（擇一）
- **資料庫**：MySQL / PostgreSQL  

系統透過 RESTful API 進行資料交換，確保系統具備良好的維護性與擴充性。
