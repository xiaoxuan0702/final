## 一、前言（Introduction）

隨著大專院校學生社團的多元發展，社團數量與活動規模逐年增加，傳統以紙本、通訊軟體或零散工具進行管理的方式，已逐漸無法滿足現代社團在成員管理、活動規劃與資訊整合上的需求。社團幹部在管理過程中，常面臨資料分散、資訊不透明以及行政效率低落等問題。因此，本專題以「學生社團管理系統」為主題，規劃並設計一套以 Web 為基礎的管理平台，提供社團成員、幹部與管理者一個集中且系統化的管理工具。透過本系統，使用者可以更方便地瀏覽社團資訊、參與活動，而社團幹部也能有效進行成員與活動管理。

本系統的目標在於提升社團行政效率、降低管理負擔，並促進校園社團運作的數位化與現代化。

---

## 二、系統概述（System Overview）


本系統為一套以 Web 為基礎的學生社團管理平台，目的在於整合校園內各類社團的管理需求，提供一個集中、便利且易於操作的數位化系統。透過瀏覽器即可使用，無需額外安裝應用程式，讓學生與社團幹部能隨時隨地進行社團相關操作。
系統主要使用對象包含以下三類角色：

- **一般學生（社團成員）**  
  可透過系統瀏覽社團列表、查看社團介紹與活動資訊，並進行社團加入申請與活動報名，清楚掌握自身的社團參與狀況。

- **社團幹部**  
  負責社團日常管理工作，包括成員審核、公告發布、活動建立與管理，以及活動報名名單的查看，藉此降低行政負擔並提升管理效率。

- **系統管理員**  
  擁有最高管理權限，負責維護系統運作、管理使用者帳號與社團資料，確保系統資料的正確性與安全性。

整體系統以「角色權限區分」為核心設計概念，不同身分的使用者僅能操作對應權限的功能，以確保資料安全與管理流程的清楚分工。透過本系統，校園社團的資訊管理將更加有系統性，並促進社團運作的效率與透明度。

## 三、功能需求分析（Functional Requirements）
本系統依使用者身分不同，將功能分為「一般使用者」、「社團幹部」與「系統管理員」三大類，透過明確的角色權限區分，確保系統操作流程清楚且資料安全。

### 1、使用者功能（一般學生）
一般學生為系統的主要使用者，其功能需求如下：

- **使用者註冊與登入**  
  學生可透過系統進行帳號註冊與登入，建立個人帳戶，以便使用後續社團相關功能。

- **瀏覽社團列表與社團詳細資訊**  
  使用者可查看所有社團的基本資料，包括社團介紹、成立宗旨、幹部資訊與活動紀錄，協助學生選擇適合的社團。

- **申請加入社團**  
  使用者可向有興趣的社團提出加入申請，並等待社團幹部審核。

- **報名社團活動**  
  使用者可瀏覽社團所舉辦的活動，並在線上完成活動報名，減少紙本或人工統計的困擾。

- **查詢個人參與紀錄**  
  使用者可查看自己所加入的社團與已報名或參與過的活動紀錄，方便進行個人管理。


### 2、社團幹部功能
社團幹部負責社團的日常運作與管理，其功能需求包含：

- **管理社團成員**  
  幹部可審核學生的入社申請、移除成員或調整成員身分，確保社團成員名單的正確性。

- **發布社團公告**  
  幹部可發布社團公告，向成員傳達重要訊息，如活動通知、會議時間或社團事項。

- **建立與管理社團活動**  
  幹部可建立活動資料，包含活動名稱、時間、地點與說明，並可進行活動內容的修改與管理。

- **查看活動報名名單**  
  幹部可即時查看活動報名人數與名單，方便進行活動規劃與人數控管。

### 3、系統管理員功能
系統管理員為整體系統的最高管理角色，其功能需求如下：

- **管理使用者帳號**  
  管理員可查看、停用或刪除使用者帳號，維護系統使用秩序。

- **管理社團資料**  
  管理員可新增、修改或刪除社團資料，確保系統內社團資訊的正確性與完整性。
  
- **系統權限控管與維護**  
  管理員負責系統權限設定、資料維護與系統正常運作，確保系統安全性與穩定性。

透過上述功能設計，本系統能有效支援學生社團的日常管理需求，提升行政效率，並提供使用者良好的操作體驗。

## 四、系統設計🎨 （System Design）
本系統採用「前後端分離（Frontend / Backend Separation）」的系統架構設計，透過明確的模組分工，提升系統的維護性、擴充性與開發效率。

### 1、系統架構說明
系統主要分為三個核心層級：

- **前端（Frontend）**
- **後端（Backend）**
- **資料庫（Database）**

各層級透過 RESTful API 進行資料交換，彼此獨立運作，降低系統耦合度。


### 2、前端設計（Frontend）
前端負責使用者介面呈現與操作互動，主要技術包含：

- HTML：負責頁面結構
- CSS：負責畫面樣式與版面配置
- JavaScript：負責互動邏輯與資料請求

前端透過瀏覽器向後端 API 發送請求，取得社團、活動與使用者相關資料，並即時更新畫面內容，以提供良好的使用者體驗。

### 3、後端設計（Backend）
後端負責處理系統核心邏輯與資料存取，技術可選擇以下其中一種實作：

- Node.js
- PHP
- Python

後端主要功能包含：
- 使用者身份驗證與權限控管
- 社團、成員與活動資料管理
- API 服務提供（RESTful API）

後端接收前端請求後，進行資料處理並回傳結果，確保系統運作穩定且安全。

### 4、資料庫設計（Database）
系統資料儲存採用關聯式資料庫管理系統，如：

- MySQL
- PostgreSQL

資料庫負責保存使用者、社團、成員關係與活動資訊，透過良好的資料表設計，確保資料一致性與完整性。

### 5、API 與資料交換機制
系統採用 RESTful API 作為前後端溝通方式，具有以下優點：

- 標準化資料傳輸格式（JSON）
- 提升系統可讀性與維護性
- 方便未來擴充行動裝置或其他系統整合

### 6、系統設計優點

- 前後端分離，便於多人協作開發
- 架構清楚，降低維護與修改成本
- 易於擴充新功能與新平台
- 提升系統穩定性與安全性

透過上述系統設計，本專題能有效支援學生社團管理需求，並具備未來功能擴充與系統升級的彈性。

## 五、資料庫設計（Database Design）
本系統採用關聯式資料庫（Relational Database）進行資料儲存，透過良好的資料表設計來確保資料的一致性、完整性與可維護性。各資料表之間以主鍵（Primary Key）與外鍵（Foreign Key）建立關聯，以支援學生社團管理系統的各項功能需求。

### 1、Users（使用者）

此資料表用於儲存系統中所有使用者的基本資料，包含一般學生、社團幹部與系統管理員。

**欄位說明：**
- `user_id`：使用者唯一識別碼（Primary Key）
- `name`：使用者姓名
- `email`：使用者電子郵件（作為登入帳號）
- `password`：使用者密碼（加密後儲存）
- `role`：使用者身分（student / leader / admin）

此資料表為系統核心資料表之一，其他資料表會透過 `user_id` 與其建立關聯。

### 2、Clubs（社團）
此資料表用於儲存各學生社團的基本資訊。

**欄位說明：**
- `club_id`：社團唯一識別碼（Primary Key）
- `club_name`：社團名稱
- `description`：社團介紹與宗旨
- `leader_id`：社團負責人（Foreign Key，對應 Users.user_id）

透過 `leader_id` 欄位，系統可辨識社團幹部並賦予相應的管理權限。

### 3、Club_Members（社團成員）
此資料表為使用者與社團之間的關聯表，用於記錄學生加入社團的情況。

**欄位說明：**
- `id`：資料唯一識別碼（Primary Key）
- `club_id`：社團編號（Foreign Key，對應 Clubs.club_id）
- `user_id`：使用者編號（Foreign Key，對應 Users.user_id）
- `join_date`：加入社團日期

此設計可支援一位使用者加入多個社團，同時也方便查詢社團成員名單。

### 4、Activities（活動）
此資料表用於儲存社團所舉辦的各項活動資訊。

**欄位說明：**
- `activity_id`：活動唯一識別碼（Primary Key）
- `club_id`：所屬社團編號（Foreign Key，對應 Clubs.club_id）
- `title`：活動名稱
- `date`：活動日期
- `description`：活動內容說明

透過 `club_id`，系統可將活動與對應社團進行關聯，方便管理與查詢。

### 5、資料庫設計特色

- 採用關聯式資料表設計，確保資料一致性
- 使用外鍵建立表與表之間的關聯關係
- 支援多社團、多成員與多活動管理
- 方便後續功能擴充，如活動報名、簽到紀錄等

透過上述資料庫設計，本系統能有效支援學生社團管理的各項資料需求，並具備良好的擴充與維護彈性。

## 六、介面設計（UI Mockup）
本系統之介面設計以「簡潔、直覺、易操作」為核心原則，讓不同角色的使用者皆能快速上手操作。整體視覺風格以「校園科技感」為主，搭配清楚的版面配置與一致的色彩設計，以提升使用體驗與系統辨識度。

### 1、登入／註冊頁面

登入與註冊頁面為使用者進入系統的第一個介面，設計重點如下：
**提供清楚的帳號註冊與登入欄位**
- 顯示系統名稱與簡要說明
- 提供錯誤提示與驗證訊息（如帳號或密碼錯誤）
- 介面簡潔，避免過多干擾元素

此頁面確保使用者能快速完成登入或建立帳號，順利進入系統。


### 2、社團列表頁
社團列表頁提供所有社團的總覽資訊，功能與設計包含：

此頁面讓學生能清楚了解校園內所有社團的概況。


### 3、社團詳細頁

社團詳細頁用於呈現單一社團的完整資訊，內容包含：

活動管理頁主要提供社團幹部使用，其設計重點如下：

此頁面讓使用者能清楚掌握自身在系統中的所有紀錄。

### 6、整體介面風格說明

整體介面設計特色如下：
- 採用一致的色彩與版面配置
- 以校園科技感為主要視覺風格

- 提升使用者整體操作體驗

透過良好的介面設計，本系統能讓使用者在進行社團相關操作時更加直覺與便利。

## 七、系統特色⭐ 
本學生社團管理系統針對校園社團實際運作需求進行設計，具備多項實用且具擴充性的系統特色，能有效提升社團管理效率與使用體驗。

### 1、集中管理社團與活動資訊
系統將社團基本資料、成員名單、公告與活動資訊集中於同一平台進行管理，避免資訊分散於不同工具或平台。使用者可透過單一系統快速查詢所需資訊，提升資料管理的便利性與一致性。


### 2、明確的角色與權限區分
系統依使用者身分區分為一般學生、社團幹部與系統管理員三種角色，不同角色擁有對應的操作權限。此設計可確保資料安全性，並避免未授權的操作，同時也讓管理流程更加清楚明確。

### 3、提升社團行政管理效率
透過線上化的成員管理、活動建立與報名功能，社團幹部可大幅減少紙本作業與人工統計的時間成本，讓社團行政作業更加快速且有效率。

### 4、支援多社團與多活動管理
系統支援多個社團同時使用，每個社團皆可獨立管理自身成員與活動資訊。此外，系統也可同時處理多場活動的建立與報名需求，適用於社團數量與活動規模較大的校園環境。

### 5、架構具備良好擴充性
本系統採用前後端分離架構，並透過 RESTful API 進行資料交換，未來可依需求彈性擴充功能，例如加入行動裝置支援、活動簽到系統或通知推播等，具備良好的發展潛力。


透過上述系統特色設計，本學生社團管理系統能有效滿足校園社團管理需求，並提供穩定且具未來發展性的數位化管理平台。

使用者可透過系統進行：
**查詢社團資訊**
**參與及報名社團活動**
**管理社團事務**

系統採模組化設計，讓不同角色依權限使用不同功能，確保資料安全並提升系統維護性。

🔹 前端
HTML / CSS / JavaScript  
🔹 後端（擇一）
Node.js
PHP
Python  
🔹 資料庫

## 🗃️ 資料庫設計（Database Design）

### 📍 Users（使用者）
| 欄位名稱     | 說明                 |
| -------- | ------------------ |
| user_id  | 使用者編號（Primary Key） |
| name     | 姓名                 |
| email    | 電子郵件（登入帳號）         |
| password | 密碼（加密儲存）           |
| role     | 角色（學生 / 幹部 / 管理員）  |

### 📍 Clubs（社團)
| 欄位名稱        | 說明                |
| ----------- | ----------------- |
| club_id     | 社團編號（Primary Key） |
| club_name   | 社團名稱              |
| description | 社團介紹與宗旨           |
| leader_id   | 幹部編號（對應 Users）    |

### 📍 Club_Members（社團成員）
| 欄位名稱      | 說明                |
| --------- | ----------------- |
| id        | 資料編號（Primary Key） |
| club_id   | 所屬社團編號（FK）        |
| user_id   | 成員編號（FK）          |
| join_date | 加入日期              |

### 📍 Activities（活動）
| 欄位名稱        | 說明                |
| ----------- | ----------------- |
| activity_id | 活動編號（Primary Key） |
| club_id     | 主辦社團編號（FK）        |
| title       | 活動名稱              |
| date        | 活動日期              |
| description | 活動內容說明            |
